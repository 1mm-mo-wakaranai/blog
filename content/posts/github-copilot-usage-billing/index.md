## GitHub Copilotの課金体系が大きく変わる

2026年6月1日から、GitHub Copilotの課金体系が変わります。
これまでの月額固定制から、使用量ベース課金に移行します。
この変更は、すべてのCopilotユーザーに影響を与えます。

GitHub Copilot（AIによるコード補完ツール）は多くの開発者に愛用されています。
コードの自動補完、チャットによる質問、テスト生成などが可能です。
その課金方法が根本的に変わるため、事前の理解が重要です。

### 変更の背景

GitHubが使用量ベース課金に移行する理由は明確です。
AI処理にはモデルの推論コストがかかります。
ヘビーユーザーとライトユーザーで、コストに大きな差があるのです。

月額固定制では、あまり使わないユーザーが割高になっていました。
逆に、大量に使うユーザーは実質的に割安でした。
使用量ベースにすることで、公平な料金体系を実現します。

## 現在の料金体系との違い

まず、現在の料金体系を確認しましょう。
その上で、新しい使用量ベース課金との違いを見ていきます。

### 現在の月額固定プラン

| プラン | 月額料金 | 主な機能 |
|--------|---------|---------|
| Free | $0 | 月50回のチャット、限定的なコード補完 |
| Individual | $10 | 無制限のコード補完、チャット |
| Business | $19 | チーム管理、ポリシー設定 |
| Enterprise | $39 | 高度なカスタマイズ、専用モデル |

### 新しい使用量ベース課金

新しい課金体系では、トークン消費量に基づいて料金が決まります。
基本料金＋従量課金のハイブリッド方式です。

| プラン | 基本料金 | 含まれるトークン | 超過料金 |
|--------|---------|----------------|---------|
| Free | $0 | 月5万トークン | 利用不可 |
| Pro | $4 | 月50万トークン | $0.01/1000トークン |
| Business | $12 | 月200万トークン | $0.008/1000トークン |
| Enterprise | $25 | 月1000万トークン | $0.005/1000トークン |

### 料金シミュレーション

一般的な開発者の使用量で比較してみましょう。

- ライトユーザー（月20万トークン）：旧$10 → 新$4（60%削減）
- 標準ユーザー（月50万トークン）：旧$10 → 新$4（変わらず）
- ヘビーユーザー（月150万トークン）：旧$10 → 新$14（40%増加）

ライトユーザーにとっては値下げ、ヘビーユーザーには値上げとなります。

## トークンとは何か ― 初心者向け説明

「トークン」という言葉が出てきました。
これはAI分野の基本概念なので、しっかり理解しておきましょう。

### トークンの基本

トークンとは、AIがテキストを処理する際の最小単位です。
日本語の場合、1文字が1〜3トークンに相当します。
英語の場合、1単語が1〜2トークンに相当します。

たとえば「こんにちは」は約3〜5トークンです。
「Hello, world!」は約4トークンです。
プログラミングコードも同様にトークンに分割されます。

### Copilotでのトークン消費

GitHub Copilotでは、以下の操作でトークンが消費されます。

- コード補完の提案を受ける（入力＋出力のトークン）
- チャットで質問する（質問＋回答のトークン）
- コードの説明を求める（コード＋説明のトークン）
- テストコードを生成する（元コード＋テストのトークン）

重要なのは、入力と出力の両方でトークンが消費される点です。
長いコードを貼り付けて質問すると、入力だけで多くのトークンを使います。

### トークン消費量の目安

日常的な開発作業でのトークン消費量の目安です。

- コード補完1回：約100〜500トークン
- チャット1往復：約500〜2000トークン
- ファイル全体の説明：約1000〜5000トークン
- テスト生成1回：約1000〜3000トークン

1日8時間の開発で、平均2〜5万トークンを消費します。
月20営業日で計算すると、40〜100万トークンが目安です。

## VS Code 1.118のトークン効率改善

朗報があります。
VS Code 1.118で、Copilotのトークン効率が大幅に改善されました。
使用量ベース課金への移行を見据えた最適化です。

### 93%キャッシュ再利用

最大の改善点は、93%のキャッシュ再利用率です。
キャッシュとは、一度計算した結果を保存して再利用する仕組みです。
同じようなコードパターンでは、新たなトークン消費なしで補完が提供されます。

具体的には、以下の場面でキャッシュが効きます。

- 同じファイル内での類似パターンの補完
- プロジェクト内で繰り返し使われる定型コード
- 以前に生成したコードと類似する補完

これにより、実質的なトークン消費量が大幅に削減されます。

### ツール検索ツール（Tool Search Tool）

VS Code 1.118では「ツール検索ツール」も導入されました。
これは、Copilotが最適なツールを自動で選択する機能です。
不要なコンテキスト（文脈情報）の送信を防ぎ、トークンを節約します。

従来は、関連するファイルをすべてコンテキストとして送信していました。
新機能では、本当に必要な情報だけを選別して送信します。
これだけで、1回あたりのトークン消費が30〜50%削減されます。

### その他の効率化機能

VS Code 1.118には、他にも効率化機能が含まれています。

- インクリメンタル補完（差分のみを送信）
- スマートコンテキスト選択（関連度の高いファイルのみ参照）
- バッチ処理の最適化（複数の補完リクエストをまとめる）

## 無料プランへの影響

無料プランのユーザーが最も気になるのは「制限が変わるのか」でしょう。
結論から言うと、無料プランにも変更があります。

### 現在の無料プラン

現在の無料プランでは、月50回のチャット制限があります。
コード補完は限定的ながら利用可能です。
GPT-4oベースのモデルが使用されています。

### 新しい無料プラン

新しい無料プランでは、回数制限からトークン制限に変わります。
月5万トークンが上限となります。

これは実質的にどのくらいの利用量でしょうか。

- チャット約25〜50回分に相当
- コード補完約100〜500回分に相当
- 1日あたり約2500トークン（営業日ベース）

従来の月50回チャット制限と比べると、使い方次第で増減します。
短い質問を多くする人は有利、長い質問をする人は不利になります。

### 無料プランを最大限活用するコツ

限られたトークンを有効に使うためのコツをお伝えします。

- 質問は簡潔に、要点だけを伝える
- 不要なコードをコンテキストに含めない
- 補完が不要な場面ではCopilotを一時停止する
- 月の後半でトークンが足りなくなったら翌月まで待つ

## 節約のコツ ― トークン消費を抑える実践テクニック

使用量ベース課金では、トークンの節約が直接コスト削減につながります。
以下の実践テクニックを活用してください。

### プロンプトを短くする

チャットでの質問は、できるだけ短く的確にしましょう。
長い前置きや背景説明は、トークンを無駄に消費します。

```
# 悪い例（トークン消費が多い）
「私はReactを使ったWebアプリケーションを開発しています。
現在、ユーザー認証機能を実装しようとしているのですが、
JWTトークンの有効期限が切れた場合のリフレッシュ処理について
教えていただけますか？」

# 良い例（トークン消費が少ない）
「React + JWTのリフレッシュトークン実装例を教えて」
```

### 不要な拡張機能を無効化する

VS Codeの拡張機能の中には、Copilotのコンテキストを増やすものがあります。
使っていない拡張機能は無効化しましょう。

特に以下の拡張機能は、トークン消費に影響する場合があります。

- 大量のファイルをインデックスする拡張機能
- AIアシスタント系の重複する拡張機能
- コード分析を常時実行する拡張機能

### ワークスペース設定の最適化

Copilotの設定を最適化することで、トークン消費を抑えられます。

```json
{
  "github.copilot.advanced": {
    "inlineSuggestCount": 1,
    "listCount": 3
  },
  "github.copilot.enable": {
    "markdown": false,
    "plaintext": false
  }
}
```

上記の設定では、以下の最適化を行っています。

- インライン提案を1つに制限（デフォルトは3つ）
- 候補リストを3つに制限
- MarkdownやプレーンテキストではCopilotを無効化

### その他の節約テクニック

- 大きなファイルを開いたままにしない（コンテキストに含まれる）
- 「Explain」より「Fix」を使う（出力が短い）
- 生成されたコードは早めに確定する（再生成を防ぐ）
- オフライン作業時はCopilotを停止する

## 企業・チームでの対応策

企業やチームでCopilotを利用している場合、追加の対応が必要です。
予算管理とガバナンスの観点から整理します。

### 予算管理のポイント

使用量ベース課金では、月々のコストが変動します。
以下の対策で予算を管理しましょう。

- 月間トークン上限をチーム単位で設定する
- 週次でトークン消費量をモニタリングする
- 高消費メンバーには個別にアドバイスする
- 予算超過時のアラートを設定する

### ガバナンス設定

Business・Enterpriseプランでは、管理者が以下を設定できます。

- メンバーごとのトークン上限
- 利用可能な機能の制限（チャットのみ、補完のみなど）
- 利用時間帯の制限
- 特定リポジトリでのみ有効化

## よくある質問（FAQ）

### Q1. 6月1日に突然切り替わるのですか？

いいえ、移行期間が設けられています。
6月1日から新課金体系が適用されますが、最初の1ヶ月は移行猶予期間です。
7月1日までは旧料金と新料金の低い方が適用されます。

### Q2. 現在のIndividualプラン（$10/月）はなくなりますか？

はい、Individualプランは廃止されます。
代わりにProプラン（$4/月＋従量課金）に移行します。
多くのユーザーにとっては実質的な値下げとなる見込みです。

### Q3. トークン消費量はどこで確認できますか？

GitHubのSettings → Copilot → Usageで確認できます。
日別・週別・月別のトークン消費量がグラフで表示されます。
VS Code内からも、ステータスバーで現在の消費量を確認できます。

### Q4. チームで使っている場合、個人ごとに上限を設定できますか？

はい、Business・Enterpriseプランでは個人ごとの上限設定が可能です。
管理者がGitHub Organizationの設定画面から設定できます。
上限に達したメンバーには通知が送られ、翌月まで利用が制限されます。

### Q5. 学生向けの無料プラン（GitHub Education）はどうなりますか？

GitHub Educationの特典は継続されます。
学生向けには月20万トークンが無料で提供される予定です。
これは一般の無料プラン（5万トークン）の4倍の量です。
