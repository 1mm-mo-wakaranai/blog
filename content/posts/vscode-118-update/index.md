## VS Code 1.118の概要

VS Code 1.118は2026年4月29日にリリースされました。今回のアップデートの目玉は、GitHub Copilotのエージェント機能の大幅強化です。

主な変更点を一覧にまとめます。

| カテゴリ | 新機能 |
|---|---|
| Copilot | エージェントワークフローの拡張 |
| Copilot | CLIセッションのリモート制御 |
| 検索 | セマンティックコードベース検索 |
| 企業向け | エンタープライズコントロール強化 |
| 分析 | チャット履歴インサイト（Chronicle） |
| 効率化 | トークン使用量の削減 |
| UI | Webviewの改善 |
| 言語 | TypeScript 7サポート |
| 接続 | リモート開発の改善 |

VS Codeの基本操作に不安がある方は、先に[VS Codeのショートカット10選](/posts/vscode-shortcuts-beginner/)を確認しておくと理解しやすくなります。

## 注目の新機能

### Copilotエージェントワークフローの拡張

今回最も大きな変化は、Copilotの「エージェント機能」が強化されたことです。

エージェント機能とは、AIが1つの指示から複数の作業を自律的にこなす仕組みです。たとえば「このバグを直して」と指示するだけで、以下を自動で実行します。

1. 関連するコードを検索
2. 原因を特定
3. 修正コードを生成
4. テストを実行して確認

1.118では、このエージェントが使えるツール（道具）が増えました。具体的には以下が追加されています。

- **ツール検索ツール**: 常時読み込むツールを約30個に絞り、残りは必要なときだけ動的に読み込む
- **エージェント検索ツール**: コード検索を専用の小型モデルに委任して高速化
- **エージェント実行ツール**: ターミナルコマンドの実行を専用モデルに委任

これにより、エージェントがより賢く、より速く作業をこなせるようになりました。

GitHub Copilotの基本的な使い方は[GitHub Copilot無料プランの始め方](/posts/github-copilot-free/)で解説しています。

### CLIセッションのリモート制御

Copilot CLIで長時間かかるタスクを実行中、PCの前を離れなければならない場面があります。これまでは、AIが承認を求めて止まっても、戻るまで作業がストップしていました。

1.118では、GitHub.comやGitHubモバイルアプリから、進行中のCopilot CLIセッションを監視・操作できるようになりました。

#### 有効にする方法

1. VS Codeの設定で `github.copilot.chat.cli.remote.enabled` を有効にする
2. Copilot CLIのチャットで `/remote on` と入力する

これで外出先からスマホでCopilotの進捗を確認し、承認や指示を出せます。

#### こんな場面で便利

- 長時間かかるタスクをCopilotに任せて離席するとき
- 移動中にスマホから進捗を確認したいとき
- 承認待ちで止まっているタスクをすぐに再開したいとき

現時点では実験的機能（Experimental）の扱いです。安定版になるまでに仕様が変わる可能性があります。

### セマンティックコードベース検索

これまでGitHubやAzure DevOpsのリポジトリでしか使えなかった「セマンティック検索」が、すべてのワークスペースで使えるようになりました。

#### セマンティック検索とは

通常の検索は、入力した文字列と完全一致するコードしか見つけられません。セマンティック検索は「意味」で検索します。

たとえば「ユーザー認証の処理はどこ？」と聞くと、コード内に「認証」という文字がなくても、`login`、`signIn`、`verifyCredentials` といった関連コードを見つけてくれます。

#### 使い方

特別な設定は不要です。Copilotに質問するだけで自動的にセマンティック検索が使われます。

初回はインデックス（検索用の索引）の構築に数分かかります。手動で構築したい場合は、コマンドパレット（`Ctrl+Shift+P`）から「Build Codebase semantic index」を実行してください。

#### 初心者にとっての嬉しさ

大きなプロジェクトで「この処理どこに書いてあるんだろう？」と迷ったとき、日本語で聞くだけで該当箇所を見つけてくれます。コードの全体像を把握するのに非常に役立ちます。

### TypeScript 7サポート

TypeScript 7.0は、ネイティブコード（Go言語）で完全に書き直されたバージョンです。パフォーマンスが劇的に向上しています。

VS Code自体の開発ビルドもTypeScript 7を使うようになり、型チェック（コードの整合性確認）が60秒→10秒に短縮されました。

試してみたい方は、VS Codeの拡張機能「TypeScript Native preview」をインストールするだけです。TypeScriptの基本については[TypeScript入門ガイド](/posts/typescript-beginner/)で解説しています。

### エンタープライズ向けコントロール強化

企業のIT管理者向けの機能も強化されました。

- Copilotが参照するコンテキスト（情報源）を管理者が制限できる
- 特定のツールやMCPサーバーの使用を組織単位で制御できる
- セキュリティポリシーに沿ったCopilotの運用が可能に

個人ユーザーには直接関係しませんが、会社でVS Codeを使っている方は、管理者の設定によって使える機能が変わる場合があります。

### チャット履歴インサイト（Chronicle）

Copilotとのチャット履歴を活用する新機能「Chronicle」が追加されました。ローカルのSQLiteデータベースにチャット履歴を記録し、後から検索・要約できます。

| コマンド | 機能 |
|---|---|
| `/chronicle:standup` | 過去24時間の作業をレポートとして生成 |
| `/chronicle:tips` | 7日間の使い方を分析して改善ヒントを提示 |
| `/chronicle [質問]` | 「昨日どのファイルを編集した？」など自由に質問 |

設定で `github.copilot.chat.localIndex.enabled` を有効にすると使えます。

### トークン使用量の削減

2026年6月1日からGitHub Copilotが使用量ベースの課金に移行します。それに合わせて、トークン（AIへの入力量の単位）の効率が大幅に改善されました。

主な改善点は以下の通りです。

- **プロンプトキャッシュの最適化**: リクエストの93%以上がキャッシュから再利用
- **ツールの動的読み込み**: 必要なツールだけを読み込み、最大20%のトークン節約
- **WebSocket対応**: OpenAIモデルが12%高速化

ユーザー側で設定は不要です。アップデートするだけで自動的に効率が改善されます。

### Webviewの改善

VS Code内のWebview（拡張機能が表示するWebページ）のパフォーマンスが改善されました。拡張機能の表示が速くなり、メモリ使用量も削減されています。

### リモート開発の改善

SSH接続やDev Container（開発用コンテナ）での作業がより安定しました。接続が切れた際の再接続も改善されています。

## 初心者にとって何が嬉しいのか

新機能が多くて「結局自分に関係あるの？」と思うかもしれません。初心者にとって特に嬉しいポイントを3つに絞ります。

### 1. コードの場所を日本語で探せる

セマンティック検索により、「ログイン処理はどこ？」と日本語で聞くだけで該当コードが見つかります。大きなプロジェクトで迷子にならなくなります。

### 2. AIがより賢く手伝ってくれる

エージェント機能の強化により、「このエラーを直して」と言うだけでAIが原因特定から修正まで自動でやってくれます。初心者がエラーで詰まる時間が大幅に減ります。

### 3. 動作が速くなった

トークン効率の改善とWebviewの最適化により、VS Code全体の動作が軽くなっています。低スペックのPCでも快適に使えるようになりました。

AIを活用したコーディングに興味がある方は、[AIコーディングアシスタント比較](/posts/ai-coding-assistant/)も参考にしてください。

## アップデート方法

VS Codeのアップデートは通常自動で行われます。手動で確認・実行する方法は以下の通りです。

### Windows / Linuxの場合

1. メニューバーの「ヘルプ」をクリック
2. 「更新の確認」を選択
3. アップデートがあれば「再起動して更新」をクリック

### macOSの場合

1. メニューバーの「Code」をクリック
2. 「更新の確認」を選択
3. アップデートがあれば「再起動して更新」をクリック

### バージョンの確認方法

「ヘルプ」→「バージョン情報」で現在のバージョンを確認できます。`1.118.x` と表示されていれば最新です。

### アップデートできない場合

会社のPCなど、管理者によって自動更新が無効にされている場合があります。その場合はIT部門に相談してください。

個人PCで更新できない場合は、[VS Code公式サイト](https://code.visualstudio.com/)から最新版をダウンロードして上書きインストールする方法もあります。

## Copilotのコミット帰属問題について

1.118で導入された「Git AI共著者の自動追加」機能が、リリース直後に議論を呼びました。

### 何が起きたのか

Copilotがファイルを変更した場合、コミットメッセージに自動的にCopilotが共著者（co-author）として追加される機能が実装されました。設定名は `git.addAICoAuthor` です。

### なぜ炎上したのか

開発者コミュニティから以下の懸念が上がりました。

- コミット履歴が汚れる（毎回AIの名前が入る）
- AIの貢献度が曖昧（1文字の補完でも共著者になるのか）
- オープンソースプロジェクトでのライセンス問題
- 企業のコードレビュープロセスへの影響

特にオープンソース界隈では「AIが書いたコードの著作権は誰のものか」という根本的な問題と絡み、大きな議論になりました。

### 現在の状況

批判を受けて、OpenAI/GitHubはこの機能をデフォルトでオフに変更しました。使いたい人だけが設定で有効にする形になっています。

この問題は「AIとコードの著作権」という大きなテーマの一部です。今後もルールが変わる可能性があるので、動向を注視しておくことをおすすめします。

バイブコーディング（AIに指示してコードを書かせるスタイル）に興味がある方は、[バイブコーディングとは？初心者向け解説](/posts/vibe-coding-beginner/)もあわせてご覧ください。

## よくある質問（FAQ）

### Q: VS Code 1.118は無料で使えますか？
A: はい、VS Code自体は完全無料です。Copilot機能を使うにはGitHub Copilotのサブスクリプションが必要ですが、無料プラン（月50回のチャット）もあります。

### Q: Copilotのエージェント機能は初心者でも使えますか？
A: はい、使えます。チャット欄に日本語で「このエラーを直して」「テストを書いて」と指示するだけです。専門的なコマンドを覚える必要はありません。

### Q: セマンティック検索のインデックスはPCの容量を圧迫しますか？
A: プロジェクトの規模によりますが、通常は数十MB程度です。数GBのプロジェクトでも100MB以下に収まることがほとんどです。

### Q: TypeScript 7を使わないとVS Codeが動かなくなりますか？
A: いいえ、TypeScript 7のサポートは任意です。既存のTypeScriptプロジェクトはそのまま動きます。新しいバージョンを試したい方だけ拡張機能をインストールしてください。

### Q: 6月からのCopilot課金変更で、無料ユーザーに影響はありますか？
A: 無料プランの月50回のチャットリクエストは維持される予定です。ただし、エージェント機能の利用はトークン消費が大きいため、無料枠を早く使い切る可能性があります。使用量は設定画面で確認できます。
